10. DistanceStrategy import path: `langchain_oracledb.utils.distance_strategy` does not exist
11. ORA-43853: JSON type cannot be used in SYSTEM tablespace — skill connects as SYSTEM by default
12. verify.py error label is misleading: reports "FAIL (vector)" when check_inference() fails
13. oci-openai package incompatibility with modern openai SDK: AttributeError 'URL' object has no attribute 'decode'

## Consolidated edits applied (`d46cadbf`)

22 unique findings ⇒ 16 files changed (12 modified + 4 new).

**P0 (broken claims):**
- `oracle-database-mcp-server` not on PyPI ⇒ scaffold local LangChain BaseTool subclasses calling `oracledb.Cursor` directly.
- `oci-openai` shim broken vs modern `openai` SDK ⇒ rewrite chat factory to use `oci.generative_ai_inference.GenerativeAiInferenceClient` directly.
- ORA-43853 on JSON columns under SYSTEM tablespace ⇒ docker-setup creates an APP user with USERS tablespace + `CREATE MINING MODEL` + `EXECUTE ON SYS.DBMS_VECTOR`.
- Manual ONNX export pipeline ⇒ `onnx2oracle` CLI as default at intermediate / advanced.
- `web_fetch` tool was undefined ⇒ shipped `shared/snippets/web_fetch_tool.py` with `(url, fallback_query)` signature.

**P1 (works once you know):**
- `DistanceStrategy` import path corrected.
- `InDBEmbeddings` subclass actually shipped at `shared/snippets/in_db_embeddings.py`.
- `OracleChatHistory` hardened against `oracledb 4.x` returns; idempotent PL/SQL DDL.
- `verify.template.py` names the failing step.
- `chat_history` migration uses idempotent PL/SQL block (Oracle has no `IF NOT EXISTS`).
- env-var naming standardised on `DB_*`.
- `docker-compose.oracle-free.yml` parameterised on `ORACLE_PORT` / `ORACLE_VOLUME` / `COMPOSE_PROJECT_NAME`.
- BaseTool subclassing replaces monkeypatch (Pydantic 2).
- Advanced agent loop documents finish-or-MAX_STEPS planner rules.

**P2 (drag, not blockers):**
- Conda recipe documented.
- Docker group membership note.
- `env.example` modernised (OCI everywhere; Ollama archived).
- `onnx2oracle` CLI argument shape corrected.
- `vector_search` BaseTool docstring shows args_schema.
- Memory cleanup recipe added.

**SQLcl tee:** folded into intermediate tier by default (`shared/snippets/sqlcl_tee.py` + `shared/references/sqlcl-tee.md`).

**P3 / deferred:** ORA-22848 dev gotcha note; "notebook as Bar B" decision.
